# HiveClaw dashboard docs (Docusaurus)

Source for **`/docs`** in the Next dashboard. Builds static files into **`../public/docs`** so the app can serve them at `https://<host>/docs/`.

## Prerequisites

From the **monorepo root**, install workspace dependencies (`pnpm install`). This package is nested under `apps/hiveclaw-dashboard/docs`.

## Commands

| Goal | Command |
|------|---------|
| Edit docs with live reload (port **3041**) | `pnpm --filter hiveclaw-dashboard docs:dev` |
| Production-style build into `public/docs` | `pnpm --filter hiveclaw-dashboard docs:build` |
| Typecheck only | `pnpm --dir apps/hiveclaw-dashboard/docs run typecheck` |

From the **repo root**, **`pnpm run build`** runs **`docs:build`** first, then builds all workspace packages (the dashboard `build` is **`next build`** only). To build the dashboard package alone with fresh docs, run **`pnpm --filter hiveclaw-dashboard docs:build`** then **`pnpm --filter hiveclaw-dashboard build`**, or use **`pnpm run docs:build`** from the root before filtering the dashboard.

## Canonical URL at build time

Set **`DOCUSAURUS_URL`** when building for production (defaults to `http://localhost:3040`). Documented in the monorepo `.env.example`.

## Content

MDX lives in **`docs/docs/`**. The sidebar is autogenerated from that tree (`sidebars.ts`). After changing MDX, run **`docs:build`** (or use **`docs:dev`** for iteration only).
